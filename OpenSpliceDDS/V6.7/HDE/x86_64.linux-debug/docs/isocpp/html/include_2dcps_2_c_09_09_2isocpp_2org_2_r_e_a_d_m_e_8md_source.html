27 API documentation {#mainpage_api}
28 =================
29 
30 The namespaces below contain all of the major functionality of the ISOCPP api.
31 
32 - \subpage isocpp_dcps
33 
34  + \subpage isocpp_dcps_domain
35  + The dds::domain namespace provides facilities for creating a DomainParticipant
36  that establishes a virtual network in which an application and it's entities
37  can operate.
38  + \subpage isocpp_dcps_topic
39  + The dds::topic namespace provides facilities for the creation of Topics which
40  can be published or subscribed to, as well as providing more advanced filtering
41  through ContentFilteredTopics.
42  + \subpage isocpp_dcps_pub
43  + The dds::pub namespace provides facilities for writing samples based on a Topic.
44  + \subpage isocpp_dcps_sub
45  + The dds::sub namespace provides facilities for reading samples based on a Topic,
46  as well as providing the ability to query a read.
47  + \subpage isocpp_dcps_core
48  + The dds::core namespace provides core facilities such as QoS policies and the
49  ability to wait on particular conditions using a WaitSet.

51 Platform requirements & re-building {#mainpage_build}
52 ===================================
53 
54 The ISO C++ PSM specification requires some modern compiler features.
55 Specifically: the header file [ref_traits.hpp](@ref dds/core/ref_traits.hpp)
56 requires that shared and weak smart pointer types and functions for safe
57 polymorphism be available and are used. The header
58 [macros.hpp](@ref dds/core/ref_traits.hpp) requires the existence of a _compile
59 time_ static assert.
60 
61 To address these the OpenSplice ISO C++ DCPS API requires any one only of the
62 following:
63  + A compiler with C++ 11 support
64  + A compiler with Technical Report 1 (TR1) support
65  + The availability of [Boost]. Headers only are required.
66 
67 GCC requirements {#mainpage_gcc}
68 ----------------
69 
70 By default, the ISO C++ DCPS API can, and does, compile against the
71 TR1 templates headers available in g++ since version 4.1. Any
72 distributions built with an older version of GCC will have been compiled
73 against [Boost].
74 
75 If you would like to use your own version of [Boost] or the experimental
76 `-std=c++0x` C++11 support to develop your applications: you should rebuild the
77 ISO C++ library to suit your needs.
78 See [the rebuilding instructions](@ref isocpp_customlibs)
79 
80 Visual Studio requirements {#mainpage_vs}
81 --------------------------
82 
83 The default compilation configuration of Visual Studio distributions depends on
84 the compiler version.
85 
86 Version | Default | Optional
87 --------------------|---------|---------------
88 Visual Studio 2010 | C++11 | TR1 or [Boost]
89 Visual Studio 2008 | TR1* | [Boost]
90 Visual Studio 2005 | [Boost] | none
91 
92 * TR1 support was added to Visual Studio 2008 in Feature Pack 1. If this is not
93 installed you should use [Boost] instead.
94 
95 If you wish to change to one of the optional supprted configurations, or would
96 like to switch the API to link Debug or to your own applications version of the
97 VC++ redistributable, then you should see
98 [the rebuilding instructions](@ref isocpp_customlibs).
