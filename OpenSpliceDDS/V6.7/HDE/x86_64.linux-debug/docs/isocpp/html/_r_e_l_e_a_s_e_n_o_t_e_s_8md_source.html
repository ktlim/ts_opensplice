12 New IDL type mapping {#release_notes_newtypes640}
13 ----------------------------------------
14 Introduced the new DDS-PSM-CXX specification IDL type mapping. This allows simplified usage of IDL generated
15 types through full compatibility with the STL. All ISO C++ example code has been updated to demonstrate the new
16 usage with the exception of built-in topics (see known issues).
17 
18 The table below shows the old generated types and their equivalent in the new mapping:
19 
20 IDL Type | Old mapping | New mapping
21 ------------------------|-------------------------|------------------------
22 struct | struct | class
23 sequence<T> | IDL to C++ sequence | std::vector<T>
24 array | C++ array | dds::core::array<T, n>
25 boolean | unsigned char | bool
26 string | const char* | std::string

28 Find functions, Built-in topics {#release_notes_findandlookups640}
29 ----------------------------------------
30 Added support for obtaining the built-in Subscriber as well as the ability to find the DataReaders
31 of a Subscriber based on Topic name. This allows retrieval of the built-in DataReaders.
32 
33 - dds::sub::builtin_subscriber(dds::domain::DomainParticipant)
34  - Retrieves the built-in Subscriber for the specified domain
35 - dds::sub::find(various parameters)
36  - Retrieves a DataReader from a Subscriber with a specified Topic name. Supplying the built-in
37  subscriber will allow retrieval of the built-in DataReaders.
38 - dds::domain::find(uint32_t id)
39  - Retrieves the DomainParticipant with the specified id.
40 - dds::pub::find(various parameters)
41  - Retrieves a DataWriter from a Publisher with a specified Topic name.
42 - dds::topic::find(const dds::domain::DomainParticipant& dp, const std::string& topic_name)
43  - Retrieves a Topic from a DomainParticipant with a specified Topic name.

267 Known Issues and Currently Unsupported Features {#release_notes_6}
268 =====================================
269 
270 Although the ISO C++ DCPS API implements the majority of commonly used features, some functionality
271 of the OMG DDS specification and DDS-PSM-Cxx is not currently supported. Below is a list of known
272 issues and features that are not currently supported:
273 
274 - [Listeners](#release_notes_us_listeners)
275 - [Find and Discovery functions](#release_notes_us_find_discovery)
276 - [New IDL type mapping for built-in topics](#release_notes_us_IDLtypemapping)
277 - [OpenSplice API extensions](#release_notes_us_extensions)
278 - [Other known issues] (#release_notes_other_known_issues)
279 
280 
281 Unsupported Listeners {#release_notes_us_listeners}
282 ---------------------
283 The listeners on the following entities are not currently supported:
284 
285 - DomainParticipant
286 - Publisher
287 - Subscriber
288 - Topic
289 
290 Find and Discovery functions {#release_notes_us_find_discovery}
291 -------------------------
292 Currently, when using the find api to retrieve DataReaders or DataWriters based on their topic name,
293 only a single DataReader or DataWriter will be returned even though there may be more than one. It
294 is not specified which one will be returned. A future release will address this and allow multiple
295 DataReaders and DataWriters to be returned. As well as this the following discovery functions are
296 not currently supported:
297 
298 - [dds/domain/discovery.hpp](@ref dds/domain/discovery.hpp)
299 - [dds/topic/discovery.hpp](@ref dds/topic/discovery.hpp)
300 
301 New IDL type mapping for built-in topics {#release_notes_us_IDLtypemapping}
302 ------------------------------------
303 The _new_ mapping of primitives and container types in the DDS-PSM-CXX specification (7.4.2) is not yet implemented for built-in topics. The DataTypes
304 generated for these still follow the original OMG IDL to C++ mapping specification.
305 
306 OpenSplice API extensions {#release_notes_us_extensions}
307 -------------------------
308 The implementation contains only API features that are present in the OMG DCPS specification at this time.
309 Other OpenSplice language bindings contain extensions to the specification such as additional QoS,
310 Entity operations, and even Entities (e.g. DataReaderView). These features are not yet available.
311 
312 Other Known Issues {#release_notes_other_known_issues}
313 ------------------
314 - dds::sub::DataReader<T> >> dds::sub::DataReaderQos
315  - Attempting to extract a QoS from a DataReader using the >> operator will cause an exception.
316 
317 - dds::sub::CoherentAccess, dds::sub::CoherentSet, dds::sub::SuspendedPublication
318  - The above classes are currently not supported.
319 
320 - dds::sub::AnyDataReader::parent()
321  - The above function is not currently supported.
322 
323 - dds::sub::SharedSamples
324  - SharedSamples are not currently supported.
325 
326 - dds::core::QosProvider
327  - The QosProvider is not currently supported.
328 
329 - dds::topic::MultiTopic
330  - MultiTopics are not currently supported.
331 
332 - dds::core::policy::UserData, dds::core::policy::GroupData, dds::core::policy::TopicData
333  - The above policy classes are only partially supported. For each class, the constructor that
334  takes either two pointers or two iterators is not currently supported, nor are the begin or end
335  functions if present.
