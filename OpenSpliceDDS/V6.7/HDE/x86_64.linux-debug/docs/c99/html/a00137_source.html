898 /*
899  The "dds_waitset_wait" operation blocks until the some of the
900  attached entities has an enabled and set status condition, or
901  "reltimeout" has elapsed. The "dds_waitset_wait_until" operation
902  is the same as the "dds_wait" except that it takes an absolute timeout.
903 
904  Upon successful return, the array "xs" is filled with 0 < M <= nxs
905  values of the "x"s corresponding to the triggering entities, as
906  specified in attach_to_waitset, and M is returned. In case of a
907  time out, the return value is 0.
908 
909  Deleting the waitset while the application is blocked results in an
910  error code (i.e. < 0) returned by "wait".
911 
912  Multiple threads may block on a single waitset at the same time;
913  the calls are entirely independent.
914 
915  An empty waitset never triggers (i.e., dds_waitset_wait on an empty
916  waitset is essentially equivalent to dds_sleepfor).
